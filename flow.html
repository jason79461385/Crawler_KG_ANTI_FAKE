Scam Intel Console — 系統流程

React + Express 即時防詐 Demo · 系統架構與三大主要流程

1. 系統總覽

前端透過 REST API 進入引擎,經由 Orchestrator 分派到三個核心模組,各自呼叫對應的外部資源;Orchestrator 同時將狀態落地至持久層。

2. 資料抓取與同步

並行抓取多個來源、標準化過濾後寫入記憶體狀態,再扇出至三個持久化目的地;來源失效時切到 Demo Fallback。

3. 對話風險分析

混合評分(關鍵字 + 實體 + 語意向量),挑選最相關案例後請 LLM 生成警示;若 LLM 未配置或失敗,自動回到規則式警示。

4. 網站驗證

三條訊號管線並行匯入評分:網址表層信號、HTML 內容萃取與分析、外部威脅情資;最後彙整為單一風險判定。
